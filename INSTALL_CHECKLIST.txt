TESTING
================================================================================

STEP 8: LOAD SETTINGS
[ ] Click on your test group
[ ] Verify settings load (200+ settings)
[ ] Verify forbidden settings NOT visible:
    - additionalDiskSize
    - cores
    - cpulimit
    - cpuunits
    - diskSize
    - memory
    - vcpus

STEP 9: EXPORT SETTINGS (BACKUP)
[ ] Click "Export Current Settings to CSV"
[ ] Verify CSV downloads
[ ] Open CSV and verify data looks correct
[ ] SAVE THIS FILE as backup

STEP 10: TEST CHANGE (READ-ONLY TEST)
[ ] Find setting: userComment
[ ] Note current value
[ ] Change value to: TEST INSTALLATION
[ ] Click "Preview Changes"
[ ] Verify preview shows:
    - Product ID
    - Product name
    - Setting name: userComment
    - Old value
    - New value: TEST INSTALLATION
[ ] Click "Cancel" (DO NOT APPLY YET)

STEP 11: TEST ACTUAL CHANGE
[ ] Change userComment to: INSTALLATION VERIFIED
[ ] Click "Preview Changes"
[ ] Review preview carefully
[ ] Click "Confirm & Apply Changes"
[ ] Verify success message
[ ] Note: "Updated 1 setting(s) across 1 product(s)"

STEP 12: VERIFY CHANGE IN DATABASE
[ ] Run SQL query:
    SELECT value FROM ProxmoxVeVpsCloud_ProductConfiguration 
    WHERE product_id = [your_test_id] AND setting = 'userComment';

[ ] Verify value is: "INSTALLATION VERIFIED" (JSON-encoded)

STEP 13: VERIFY CHANGE HISTORY

[ ] Find your change
[ ] Verify shows:

EMERGENCY ROLLBACK PROCEDURE
================================================================================

If something goes wrong:

1. [ ] Stop making changes immediately
2. [ ] Review Change History to identify what was changed
3. [ ] Use CSV export to get old values
4. [ ] Manually restore old values using the addon
5. [ ] Or restore database from backup if necessary
6. [ ] Document what went wrong
7. [ ] Fix the issue before trying again
